# v0.2.0-internal.1

## Scope

First internal cut of the `v0.2.0` testnet release line from `release/testnet-v0.2.0`.

This internal release packages the smart-wallet foundation work that landed on `dev` and establishes the release-line branching, deployment, and versioning model for the `v0.2.0` cycle.

## Included Work

- Privy embedded-wallet integration with fallback-safe provider wiring
- Smart-wallet and transacting-account normalization across investor and partner flows
- Sponsored and fallback testnet funding flows
- Wallet identity, modal auth, and transaction-state UX cleanup
- Partner/admin wallet-boundary enforcement in the web app
- Release strategy aligned to the `dev -> release/* -> main` deployment model

## Deployment Model

- Integration happens on `dev`
- Release stabilization happens on `release/testnet-v0.2.0`
- Vercel production remains pinned to `main`
- Internal and beta validation should use release-branch preview deployments
- Final public testnet ship should happen by merging the approved release branch into `main`

## Validation Expectations

- Core investor flow works on `Sepolia`
- Test-funds flow works on supported mock-token testnets
- No blocker-level regressions in login, chain switching, deposit, withdraw, payout, or partner dashboard flows
- Release docs and support instructions are up to date enough for internal testers

## Known Follow-up At This Cut

- Additional internal validation is still required on `Polygon Amoy` and `Arbitrum Sepolia`
- Future partner delegation and existing-EOA migration are out of scope for this foundation release
- Public launch messaging and external release notes are not part of this internal tag
