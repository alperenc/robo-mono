# v0.2.3-internal.4

## Scope

Internal stabilization cut on top of the `v0.2.0` release line.

This release packages follow-up wallet UX fixes after the branding refresh, along with product-copy updates that better describe Roboshare as a marketplace for tokenized revenue streams.

## Included Work

- Privy reconnect flows now reopen wallet connection correctly after an authenticated session loses its active wallet context
- Standard web builds now shim Privy's optional Farcaster Solana dependency instead of failing local module resolution
- Smart-wallet transaction toasts now use neutral submission language instead of wallet-signing language
- Header, homepage, markets page, and Privy login copy now consistently describe tokenized revenue streams instead of tokenized asset trading
- Root README tagline now matches the product's revenue-stream positioning

## Validation Expectations

- Authenticated users who lose their active wallet context can reconnect from the header and modal CTAs
- Local development no longer throws on the optional `@farcaster/mini-app-solana` import path
- Transaction toasts read correctly for embedded smart-wallet flows
- Top-level product copy consistently frames claim units as exposure to revenue streams rather than direct asset-token trading

## Notes

- This is still an internal release on `release/testnet-v0.2.0`
- No protocol contracts or deployment addresses changed in this cut
