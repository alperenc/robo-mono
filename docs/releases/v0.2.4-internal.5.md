# v0.2.4-internal.5

## Scope

Internal Monad rollout cut on top of the `v0.2.0` release line.

This release adds Monad Testnet as a supported deployment target in the protocol and web app, publishes the first Monad deployment artifacts, and hardens the subgraph workflow so a fresh network deploy prepares its own ABI/codegen/build state.

## Included Work

- Monad Testnet is now recognized as a supported test network in the deploy helpers
- Scroll and Scroll Sepolia are removed from the active deploy configuration
- Web network configuration now includes Monad Testnet and its subgraph environment variable path
- The first Monad deployment artifact set is recorded for the release branch
- The web contract registry now includes Monad Testnet addresses without dropping the previously supported testnets
- Subgraph config now includes `monad-testnet` addresses and start blocks
- Root subgraph commands now prepare ABI, codegen, and build state before deploy so a fresh network does not depend on prior generated files

## Validation Expectations

- `yarn deploy:verify --network monadTestnet --keystore <keystore>` completes successfully with Monad deployment outputs
- `yarn subgraph:build --network monad-testnet` completes successfully from a fresh worktree state
- `yarn subgraph:deploy --network monad-testnet --version-label <label>` no longer fails on missing network config, missing ABIs, or missing generated types
- Web environments with `NEXT_PUBLIC_SUBGRAPH_URL_10143` set can target Monad Testnet using the generated deployment addresses

## Notes

- This is still an internal release on `release/testnet-v0.2.0`
- Monad is added as a follow-on supported testnet within the existing `v0.2.x` line rather than starting a new minor release line
